# Draft v1: Hero Heading Options

## Current hook analysis
- Current: "I break systems so they can't break you."
- What works: punchy, on-theme (offensive sec).
- What's weak: "break you" is a cliché; reads as a slogan not a statement; user rejected all 4 prior rewrites.
- Emotional impact: low — feels templated.

## Suggested alternatives (by flavor)

### Flavor A — Terminal / command line (reads as code, not copy; fits CRT hero)
- A1: `./break --target=reality`
  *Why:* looks like a shell command rendered in huge Geist; tactical, funky, zero cliché. Risk: slightly cryptic for recruiters.
- A2: `root@harrydev:~# whoami`
  *Why:* classic hacker frame; self-descriptive. Risk: over-familiar meme.
- A3: `extract --silent --now`
  *Why:* verb from the research (CH55x/firmware extraction). Risk: abstract without context.

### Flavor B — Stacked noun phrase (BEST for clamp(7vw) display type; two short lines)
- B1: `GLITCH.` / `EXTRACT.` / `REPEAT.`
  *Why:* three verbs = the funky method loop; monumental as display type; rhythm is staccato, not listy.
- B2: `SILICON` / `TO CLOUD`
  *Why:* the range of the work; two-line stack reads huge and clean.
- B3: `FINDS THE` / `WEAK SPOT`
  *Why:* active, specific; "weak spot" = vuln. Risk: slightly ad-tag.

### Flavor C — Dry statement (safest; states the work, no cleverness)
- C1: `Offensive security. Hardware hacking. Reverse engineering.`
  *Why:* literal, no metaphor, anti-slop clean. Risk: reads like a tagline list (rhythm = 3 items).
- C2: `I break hardware so software can't.`
  *Why:* keeps the "break" idea but grounds it in silicon. Risk: still a "break X so Y" structure.
- C3: `I read chips other people locked.`
  *Why:* specific to extraction/glitch work; voice-y. Risk: "read" undersells the violence of the work.

### Flavor D — Question (pulls visitor in; riskiest)
- D1: `What's inside the chip?`
- D2: `What did they think was safe?`

## Recommendation (voice-preserving, not directive)
For a 6.5rem Geist display hero, **B1 (GLITCH. EXTRACT. REPEAT.)** is the strongest:
- It's the actual funky method loop from the research (glitch → extract → repeat).
- Stacked, it fills the hero like the Awwwards "type-forward wordmark" pattern.
- Zero cliché, zero em-dash, rhythm is staccato not listy, specific to the work.
- Pairs perfectly with the existing sub-line "Offensive security, hardware hacking, autonomous AI..."

If B1 feels too abstract for recruiters, **C1** is the safe fallback and still anti-slop clean.
